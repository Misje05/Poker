# Iterasjonsplan

Denne planen er detaljert nok til at arbeidet kan fortsette og støttes av troverdige estimater.

## 1. Construction-fasen (Mot IOC)
Målet er en release som er moden nok for beta-testing i brukersamfunnet.

| Iterasjon | Oppgave | Estimat | Status |
| :--- | :--- | :--- | :--- |
| **C1** | Komplett `Evaluator`-logikk | 20 timer | Planlagt |
| **C1** | GUI for spillebord (JSP/CSS) | 15 timer | Planlagt |
| **C2** | Database-integrasjon (DAO) | 15 timer | Planlagt |
| **C2** | Samtidighet (Session management) | 15 timer | Planlagt |
| **C3** | Systemtesting og feilretting | 15 timer | Planlagt |

## 2. Transition-fasen (Mot PR)
Fokus på brukertilfredshet og at applikasjonen er ferdigstilt.

| Aktivitet | Beskrivelse                                            | Estimat |
| :--- |:-------------------------------------------------------| :--- |
| **Brukertesting** | Sikre at "Users are satisfied".                        | 10 timer |
| **Transition** | Overlevering og opplæring.                             | 5 timer |
| **Ferdigstilling** | PR-milepæl og slutt-dokumentasjon.                     | 5 timer |
